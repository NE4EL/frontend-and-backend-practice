# Практики 13–17 — Notes PWA

Веб-приложение для заметок с офлайн-режимом, push-уведомлениями и напоминаниями.

## Запуск

```bash
npm install
npm start
# → http://localhost:3001
```

---

## Архитектура

В отличие от KR2 (два отдельных сервера: React :3001 + Express :3000), здесь **один процесс** делает всё:

```
Браузер ←──HTTP──→ Express (статика: HTML, JS, иконки)
Браузер ←─Socket.IO (WebSocket)─→ Express
Браузер ←──Push API──→ Service Worker ←── web-push (сервер)
```

---

## Практика 13 — Service Worker

**Что такое:** скрипт, который браузер запускает в фоне отдельно от страницы. Не имеет доступа к DOM. Работает даже когда вкладка закрыта.

**Жизненный цикл:**
1. `register('/sw.js')` — страница регистрирует воркер
2. `install` — воркер кэширует все статические файлы
3. `activate` — удаляет старые кэши
4. `fetch` — перехватывает все сетевые запросы

**Как работает офлайн-режим:**
```
Браузер запрашивает файл
    ↓
Service Worker перехватывает запрос
    ↓
Файл есть в кэше? → да → вернуть из кэша (без сети)
                 → нет → запросить из сети
```

**Что закэшировано:** `index.html`, `app.js`, `manifest.json`, все иконки.

**Данные заметок** хранятся в `localStorage` — они живут прямо в браузере и не теряются при отключении сети.

## Практика 14 — Web App Manifest

**Что такое:** JSON-файл (`manifest.json`), который превращает сайт в PWA — браузер начинает воспринимать его как приложение.

**Что даёт манифест:**
- Кнопка «Установить» в адресной строке Chrome
- Иконка на рабочем столе / домашнем экране телефона
- Запуск в отдельном окне без адресной строки (`"display": "standalone"`)
- Цвет заголовка окна (`"theme_color"`)
- Экран-заставка при запуске на Android

**Ключевые поля:**
```json
{
  "name": "Мои заметки",
  "start_url": "/",
  "display": "standalone",
  "theme_color": "#4285f4",
  "icons": [{ "src": "icons/favicon-512x512.png", "sizes": "512x512" }]
}
```

**Иконки** — набор PNG разных размеров (16, 32, 48, 64, 128, 256, 512px). Разные платформы выбирают подходящий размер сами.

## Практика 15 — App Shell

**Что такое:** архитектурный паттерн. Интерфейс делится на две части:

| Часть | Что | Кэширование |
|---|---|---|
| **App Shell** | Каркас: шапка, навигация, кнопки | Cache First — грузится мгновенно из кэша |
| **Контент** | `content/home.html`, `content/about.html` | Network First — сначала сеть, при офлайне — кэш |

**Как работает навигация:**
```
Клик по кнопке "Главная"
    ↓
fetch('/content/home.html')  ← Service Worker перехватывает
    ↓
Сеть доступна? → да → вернуть свежий файл + сохранить в кэш
              → нет → вернуть из кэша
    ↓
innerHTML = полученный HTML  ← вставка в #app-content
    ↓
initNotes()  ← инициализация логики заметок
```

**Почему не React/Vue:** App Shell на чистом JS + fetch даёт тот же результат для учебного PWA без сборщика.

## Практика 16 — WebSocket + Push

### WebSocket (Socket.IO)

**Чем отличается от обычного HTTP:**

| HTTP | WebSocket |
|---|---|
| Клиент спрашивает — сервер отвечает | Постоянное соединение |
| Сервер не может сам отправить данные | Сервер может слать данные в любой момент |
| Каждый запрос — отдельное соединение | Одно соединение на всё время |

**Сценарий синхронизации между вкладками:**
```
Вкладка А добавляет заметку
    ↓
socket.emit('newTask', { text: '...' })  → сервер
    ↓
io.emit('taskAdded', task)  → рассылает ВСЕМ подключённым клиентам
    ↓
Вкладка Б получает событие → показывает всплывающее сообщение
```

### Push-уведомления

**Почему не просто WebSocket:** WebSocket работает только пока вкладка открыта. Push работает даже когда браузер свёрнут или вкладка закрыта — уведомление показывает ОС.

**Как это работает (три участника):**
```
1. Браузер подписывается на Push Service (серверы Google/Mozilla)
       ↓ получает объект subscription (endpoint + ключи)
2. Браузер отправляет subscription на наш сервер (POST /subscribe)
3. Наш сервер хранит subscription

Когда нужно уведомить:
Наш сервер → webpush.sendNotification(sub, payload)
                ↓
           Push Service (Google/Mozilla)
                ↓
           Service Worker → self.registration.showNotification(...)
                ↓
           Уведомление операционной системы
```

**VAPID-ключи** — пара публичный/приватный ключ. Нужны чтобы Push Service знал, что уведомление отправляет именно наш сервер, а не кто-то чужой.

```bash
npx web-push generate-vapid-keys  # генерируется один раз
```

Публичный ключ передаётся браузеру (в `app.js`), приватный остаётся только на сервере.

## Практика 17 — Детализация Push (напоминания)

**Что добавили:** заметки с запланированным временем напоминания.

**Сценарий:**
```
Пользователь вводит текст + дату/время
    ↓
addNote(text, timestamp) → localStorage
socket.emit('newReminder', { id, text, reminderTime })
    ↓
Сервер: delay = reminderTime - Date.now()
setTimeout(() => sendPushToAll(), delay)  ← хранится в Map
    ↓
Когда время пришло → push-уведомление с кнопкой «Отложить на 5 минут»
```

**Кнопка «Отложить» (snooze):**
```
Пользователь нажимает "Отложить на 5 минут" в уведомлении
    ↓
Service Worker: event.action === 'snooze'
    ↓
fetch('/snooze?reminderId=123', { method: 'POST' })
    ↓
Сервер: clearTimeout(старый) → setTimeout(новый, 5 минут)
```

**Хранение таймеров на сервере:**
```javascript
const reminders = new Map();
// ключ: id заметки
// значение: { timeoutId, text, reminderTime }
```

## Структура файлов

```
Practuce 13-17/
├── server.js          — Express + Socket.IO + web-push (всё в одном)
├── index.html         — App Shell (каркас: шапка, навигация, кнопки push)
├── app.js             — логика клиента (навигация, заметки, socket, push)
├── sw.js              — Service Worker (кэш, push-события, snooze)
├── manifest.json      — PWA манифест
├── content/
│   ├── home.html      — форма заметок + список (грузится динамически)
│   └── about.html     — страница «О приложении»
├── icons/             — PNG иконки 7 размеров
└── generate-icons.js  — скрипт генерации иконок (pure Node.js, без зависимостей)
```

## API сервера

| Метод | Путь | Описание |
|---|---|---|
| GET | `/*` | Раздача статических файлов |
| POST | `/subscribe` | Сохранить push-подписку |
| POST | `/unsubscribe` | Удалить push-подписку |
| POST | `/snooze?reminderId=N` | Отложить напоминание на 5 минут |
| GET | `/vapidPublicKey` | Получить публичный VAPID-ключ |

## Socket.IO события

| Событие | Направление | Описание |
|---|---|---|
| `newTask` | клиент → сервер | Добавлена новая заметка |
| `taskAdded` | сервер → все клиенты | Широковещательная рассылка |
| `newReminder` | клиент → сервер | Запланировать напоминание |
